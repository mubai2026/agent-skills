# Systematic Debugging · 系统性根因分析

## 四步流程

### 1. 现象复现
- 明确触发条件（双击什么、什么场景）
- 确认复现率（每次必现/偶发/概率）
- 收集现场证据（日志/截图/错误码）

### 2. 根因追溯
- 跟踪启动链路（bat → node → server → browser）
- 检查每个环节的失败处理
- 关注竞态条件和资源冲突

### 3. 方案设计
- 单实例保护（避免多个进程争抢资源）
- 健康检查（代替固定延时等待）
- 重试机制（资源冲突时的容错）
- 可视化的错误反馈

### 4. 修复验证
- 逐项确认修复点是否正确应用
- 手动测试关键场景（快速双击、端口占用等）
- 保存修复记录
